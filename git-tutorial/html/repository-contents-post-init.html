Repository Contents After Repository Initialization

  • Open the .git directory to see what is inside.
  • Since we have only initialized a basic repository not all git generated files are present in the .git directory yet, but they will appear after some use of git.

  • Now, the following files and directories should be found in the .git directory:
  • directory: hooks
  • directory: info
  • directory: objects
  • directory: refs
  • file(no extension): config
  • file(no extension): description
  • file(no extension): HEAD

  • hooks directory:
  • The place where the user can put in their client/server -side scripts.
  • Initially the directory contains sample scripts that are not in use by default.
  • No script usage is shown in this tutorial.

  • info directory:
  • Contains a global exclude file which prohibits the git repository from tracking certain files.
  • Initially ignores no files.
  • Another way to ignore files in a repository is by using .gitignore files.
  • The scope of .gitignore files are limited to the directories they are placed in.

  • objects directory: (link to "git internals" -article)
  • The actual place where data is being stored.
  • Initially contains two empty subdirectories: info, pack.
  • Additionally, as user makes commits to git repository, up to 256 additional subdirectories will be created over time, all named with 2 hexadecimal characters.
  • More about this directory structure after first commit.

  • refs directory: (link to "git internals" -article)
  • Initially contains two subdirectories: heads, tags.
  • Additionaly may contain a subdirectory: remotes. (Usually does when user pushes work to github or elsewhere.)
  • When user creates a branch, git will create a reference file in the heads directory.
  • When user creates a tag, git will create a reference file in the tags directory.
  • When user creates a remote, git will create a reference file in the remotes directory.

  • config file
  • There are three types of config files:
  • system level..
  • global level..
  • repository level..
  • ..config files. The config file in question is naturally the repository one.

  • The repository config file overwrites any config settings that are also defined in global or system level config files.

  • Username and email config settings that were set earlier in this tutorial were global level.
  • Global level config files will affect those repositories that are created by the logged-in windows user.
  • The global level config file can be found at C:\Users\<User>\.gitconfig.

  • System level config files would affect all the user accounts of the operating system.
  • The system level config file can be found at C:\ProgramData\Git\config. (ProgramData is a hidden directory).

  • description file
  • Is used to provide the name and description for the repository in a web user interface. (github.com, bitbucket.org or similar)

  • HEAD file
  • This file is a symbolic reference to the topmost commit on the branch the user is currently on.
